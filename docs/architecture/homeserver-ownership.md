# Homeserver Ownership Matrix

This matrix is the final ownership contract. Task is the public interface; paths listed here are implementations, not alternate operator interfaces. `infra:status` runs every domain status task in read-only mode.

| Component | Desired-state owner | Code owns | Runtime or operator owns | Plan/apply/status |
|---|---|---|---|---|
| OpenTofu state backend | `infra/aws/stacks/bootstrap/` | S3 bucket hardening, versioning, lock support | Emergency state-version restoration | `aws:bootstrap:plan`, `aws:bootstrap:apply`, `aws:status` |
| Route53 DNS | `infra/aws/stacks/dns/` | Hosted zone and declared homeserver records | Registrar nameserver delegation | `aws:dns:plan`, `aws:dns:apply`, `aws:status` |
| AWS foundation | `infra/aws/stacks/foundation/` | Restic bucket, IAM users/policies, access-key resources | BWS publication of generated outputs | `aws:foundation:plan`, `aws:foundation:apply`, `aws:status` |
| Atlas base OS | `infra/atlas/ansible/roles/` | Access, Docker CLI, networking, power, terminal configuration | Ubuntu package/runtime state outside declared roles | `atlas:base:plan`, `atlas:base:apply`, `atlas:status` |
| Atlas Tailscale | `infra/atlas/ansible/roles/tailscale/` | Package, service, tagged enrollment | Node identity and Tailscale control-plane lifecycle | `atlas:tailscale:plan`, `atlas:tailscale:apply`, `atlas:status` |
| Atlas workloads | `infra/atlas/ansible/roles/atlas_homeserver/` | Compose inputs, Traefik routes, Pi-hole DNS filtering, service files, backup timer | Container databases, application data, generated caches | `atlas:homeserver:plan`, `atlas:homeserver:apply`, `atlas:status` |
| Atlas Immich | `infra/atlas/ansible/roles/atlas_immich/` | Dedicated Compose inputs, NFS mount, service configuration, global Immich settings, OIDC client policy | User records and OIDC links, mobile album selection, storage labels, database contents, media, thumbnails, transcodes, model cache | `atlas:immich:plan`, `atlas:immich:apply`, `atlas:status` |
| TrueNAS datasets | `infra/truenas/tofu/` | Declared dataset tree, permissions, mount sentinel | Pool/disks, boot pool, network, one-off snapshots | `truenas:tofu:plan`, `truenas:tofu:apply`, `truenas:status` |
| TrueNAS catalog apps | `infra/truenas/apps/` | Declared apps, bounded values, managed files and path permissions | Undeclared apps, generated defaults, app runtime data | `truenas:apps:plan`, `truenas:apps:apply`, `truenas:status`; typed reconciler retained because providers lack safe catalog-app field parity |
| Audiobookshelf | `infra/truenas/apps/`, `infra/truenas/audiobookshelf/`, `infra/truenas/tofu/`, `infra/truenas/backrest/`, `infra/identity/authentik/tofu/` | Catalog values, host-path datasets, read-only library mount, root bootstrap, stable server/scanner settings, `/books` library definition and settings, built-in backup policy, native OIDC settings/provider, backup/snapshot schedules | Progress, scans, metadata edits, OIDC subject links, additional runtime users, API keys, feeds, sessions, built-in backup execution | `truenas:apps:plan`, `truenas:audiobookshelf:plan`, `identity:authentik:plan`, `truenas:backrest:plan`, `truenas:snapshots:audiobookshelf:plan`, `truenas:snapshots:books:plan` |
| TrueNAS snapshots | `infra/truenas/snapshots/` | Declared schedules on bounded datasets | One-off snapshots and tasks on other datasets | `truenas:snapshots:plan`, `truenas:snapshots:apply`, `truenas:status`; typed reconciler retained because the provider has no snapshot-task resource |
| TrueNAS disk maintenance | `infra/truenas/maintenance/` | SMART daemon boot/runtime state, existing pool scrub policy, and all-disk SMART short/extended schedules | Manual tests, test results, disk replacement, pool topology, ongoing systemd supervision | `truenas:maintenance:plan`, `truenas:maintenance:apply`, `truenas:status`; apply is blocked unless the pool is `ONLINE` |
| TrueNAS NFS | `infra/truenas/shares/` | Declared exports and service enablement | Undeclared exports and clients | `truenas:nfs:plan`, `truenas:nfs:apply`, `truenas:status`; typed reconciler retained because the provider has no NFS resources |
| TrueNAS SMB | `infra/truenas/shares/` | Declared temporary migration shares and service enablement | Undeclared shares and clients | `truenas:smb:plan`, `truenas:smb:apply`, `truenas:status`; typed reconciler retained because the provider has no SMB resources |
| Backrest policy | `infra/truenas/backrest/` | Declared repository fields, plans, schedules, explicit retirements | Repository GUID, auth, history, cache, logs, contents | `truenas:backrest:plan`, `truenas:backrest:apply`, `truenas:status`; typed reconciler retained because config CRUD is not a stable public API |
| TrueNAS Homepage API key | `infra/truenas/tofu/` and typed publisher | Key-presence contract and atomic BWS publication | Operator-managed rotation of partial/pre-existing state | `truenas:api-key:plan`, `truenas:api-key:apply`, `truenas:status`; provider-generated keys would enter state |
| Authentik | `infra/identity/authentik/tofu/` | Default brand, Catppuccin theme, users, groups, flows, providers, applications, policy bindings | Password changes, sessions, runtime reset markers | `identity:authentik:plan`, `identity:authentik:apply`, `identity:status` |
| OpenCode remote access | Chezmoi plus `infra/aws/stacks/dns/`, `infra/atlas/ansible/roles/atlas_homeserver/`, `infra/identity/authentik/tofu/`, and `infra/network/tailscale/` | Friendly DNS, Atlas Traefik route and Homepage card, narrow Atlas-to-Mac ACL, loopback launchd services, Tailscale Serve backend, OIDC provider/application, Michail-only group policy, BWS metadata | OpenCode sessions, browser cookies, Tailscale node identity, Mac sleep/login state | `aws:dns:plan`, `atlas:homeserver:plan`, `network:policy:plan`, `chezmoi apply`, `identity:authentik:plan` plus matching apply tasks |
| Tailscale ACL | `infra/network/tailscale/` | Whole HuJSON policy | Devices, routes, DNS, auth keys, node lifecycle | `network:policy:plan`, `network:policy:apply`, `network:status`; external validation remains until released provider support exists |
| Production OpenWrt router | `infra/network/openwrt/` contract plus `infra/ansible/roles/openwrt_*` | Exact WR3000E v1/R53 management image and package policy; Ansible owns routine system, trusted DHCP/DNS/reservations, ordered Pi-hole/public fallback resolvers, main Wi-Fi, SQM, and flow offload; the protected extension owns base LAN/management, VLAN 835 PPPoE, guest, firewall, and IPv6-PD | Physical cabling and rollback; leases, WAN addresses, delegated prefixes, counters, logs, host/TLS keys, root password hash, calibration/MAC data, bootloader state, and emergency LuCI drift until reconciled | `network:openwrt:plan`, `network:openwrt:apply`, `network:openwrt:status`; build/bootstrap/backup/recover/upgrade are explicit operations, and protected mutation is direct-wired/operator-run |
| Syncthing | `infra/sync/syncthing/` | Declared options, peer/folder IDs, ordered ignore files | API keys, generated device IDs, undeclared peers/folders | `sync:plan:all`, `sync:configure:all`, `sync:status`; typed reconciler retained because the collection cannot preserve unmanaged bindings and ignore ordering |
| BWS metadata and injection | `infra/secrets/` | Aliases, BWS IDs, owners, consumers, generation and rotation metadata, target bindings | Secret values and BWS access token | `infra:desired:validate`; no apply/status exists because BWS value mutation is intentionally provider/operator-specific |

OpenTofu state ownership is additionally split across the fixed keys documented in [Homeserver IaC](homeserver-iac.md#opentofu-state). No state may own resources from another key.
