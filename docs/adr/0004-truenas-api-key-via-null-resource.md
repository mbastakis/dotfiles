# Create TrueNAS API keys via null_resource instead of manual UI entry

The `deevus/truenas` Terraform provider (v0.16.0) has no `api_key` resource, but the TrueNAS middleware supports `api_key.create` via `midclt` over SSH. To keep API key creation code-managed rather than a manual UI step, the truenas stack uses a `null_resource` with a `local-exec` provisioner that invokes the typed `homeserver-iac api-key` publisher over the existing SSH bootstrap path. The publisher verifies the committed BWS project/key metadata, checks both remote systems, creates only when both sides are absent, and stores the returned value in BWS for the Homepage dashboard widget. If BWS publication fails, it removes only the TrueNAS key created by that invocation; partial pre-existing state requires operator-managed rotation. The generated value is handled transiently and is not intentionally persisted in OpenTofu state; the resource stores only non-secret metadata and has `prevent_destroy`. If the provider later adds a native `api_key` resource, the `null_resource` can be swapped with no downstream changes.
