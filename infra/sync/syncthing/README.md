# Syncthing

Code-managed Syncthing configuration for homeserver Obsidian sync.

## Owns

- Folder definitions (ID, label, path, type, versioning)
- Device pairing (device IDs, addresses)
- Syncthing options (Tailscale-only: global discovery off, relay off)
- `.stignore` patterns (excludes `.obsidian/` and `.git/`)

## Does Not Own

- Syncthing app installation on TrueNAS (see `infra/truenas/apps/declarations/syncthing.yaml`)
- Syncthing installation on Mac (Brewfile)
- Device ID generation (runtime, first run)
- API keys (generated by Syncthing, read from `config.xml` at runtime)

## Tailscale-only Operation

Global discovery, NAT traversal, and relaying are disabled. Peers connect directly over Tailscale:

- **truenas** listens on `100.122.166.74:22000` (Tailscale IP) with `192.168.1.74:22000` (LAN) as fallback.
- **mac** uses `dynamic` addresses — truenas connects to it via Tailscale discovery or local announce.
- Local discovery stays enabled for LAN convenience.

Live TrueNAS reconciliation also uses local-first routing. The wrapper reads the API key over the routed `truenas` SSH alias and sets the Syncthing API URL to the verified LAN address or the Tailscale address selected by `~/bin/homeserver-route`.

## .stignore

The `.obsidian/` directory is excluded from sync because Obsidian config is managed by this dotfiles repo (chezmoi `private_Documents/notes/dot_obsidian/`). Each device should use its own Obsidian config — the phone should not inherit desktop-specific settings, plugins, or keybindings.

`.git/` is also excluded — the vault is a git repo on Mac, but git state should not sync to all devices.

## Files

| Path | Purpose |
|---|---|
| `config.yaml` | Schema v2 bounded device IDs, addresses, stable folder IDs, host paths, and Syncthing options |
| `stignore` | Ignore patterns deployed to each synced folder root |
| `configure-syncthing` | Stable thin shim over typed plan/apply |

## Commands

```bash
task sync:plan:mac              # Read-only workstation plan
task sync:plan:truenas          # Read-only TrueNAS plan
task sync:plan:all              # Plan both
task sync:configure:mac      # Configure Mac Syncthing
task sync:configure:truenas  # Configure TrueNAS Syncthing
task sync:configure:all      # Configure both
```

The Syncthing API key is read from the running instance's `config.xml` — no secrets are stored in the repo.

`config.yaml` validates offline against `infra/schemas/syncthing.schema.json`.
Run `task infra:desired:validate` without contacting Syncthing; live API access
begins only in explicit plan/configure tasks. Plans compare managed options,
the declared peer, declared folders, and `.stignore`; runtime API fields and
undeclared devices/folder bindings are preserved. In particular, the manually
paired iPhone and its TrueNAS folder binding are outside the typed ownership
boundary.

## Adding a New Device

1. Install Syncthing on the new device.
2. Get its device ID from the Syncthing UI (Actions → Show ID).
3. Add a new entry to `config.yaml` under the appropriate host key.
4. Add the device ID to the `devices` list of each shared folder.
5. Run `task sync:configure:all`.

## Adding a New Folder

1. Add a folder entry to `config.yaml` under `folders`.
2. Provide `truenas_path` (container-internal `/sync/...`), `truenas_host_path` (dataset path used for `.stignore`), and `mac_path`.
3. Run `task sync:configure:all`.
4. For truenas, create the dataset and add it to the Syncthing app declaration's `additional_storage` in `infra/truenas/apps/declarations/syncthing.yaml`.

## iPhone

iPhone sync uses Möbius Sync Pro (paid iOS Syncthing client). The iPhone device ID must be added to `config.yaml` and paired with truenas manually — the Mac configure script does not manage iPhone directly.

Workflow: Möbius Sync Pro syncs a local folder → Obsidian opens that folder → edits sync back through truenas over Tailscale.
